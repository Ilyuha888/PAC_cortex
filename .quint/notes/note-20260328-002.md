# Fix MaxLen(5→8) + non-zero thinking budget + defensive LLM fallback

## Rationale

Two root causes fixed: (1) MaxLen(5) on plan_remaining_steps_brief wasted a retry when LLM returned 6-8 items — raised to MaxLen(8). (2) thinking_budget:0 caused choices:null on complex structured outputs via litellm proxy (t18/t19 crashes). Set to configurable 1024 tokens (LLM_THINKING_BUDGET env var). Added defensive fallback: strip extra_body on final TypeError retry. Added RuntimeError catch around parse_step for graceful OUTCOME_ERR_INTERNAL instead of crash. Alternative: remove thinking_config entirely — rejected because unconstrained thinking burns ~490 tokens/call unnecessarily.

## Evidence

Benchmark: 22/22 (100.0%) — up from 20/22 (90.9%). t18 and t19 both pass. No MaxLen retries in logs. Total tokens: 438K prompt + 65K completion.
