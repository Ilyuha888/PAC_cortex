# Official SDK surface confirmed: exact method signatures for HarnessServiceClientSync + PcmRuntimeClientSync

## Rationale

From bitgn/sample-agents pac1-py. Harness: status(), get_benchmark(), start_playground() → (harness_url, trial_id, instruction), end_trial() → (score, score_detail). VM: PcmRuntimeClientSync(harness_url) — takes per-task URL. Methods: tree/find/search/list/read/write/delete/mk_dir/move/answer. All protobuf responses use MessageToDict(). ReportTaskCompletion is a local stop that calls vm.answer(). OUTCOME_BY_NAME maps string → Outcome enum. Entire stack is sync — no asyncio in sample.
