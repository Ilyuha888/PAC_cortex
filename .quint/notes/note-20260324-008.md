# PAC1-DEV live: real API is gRPC/ConnectRPC, not REST

## Rationale

BitGN released PAC1-DEV benchmark. The real transport is gRPC/ConnectRPC via `bitgn-api-connectrpc-python` (buf.build), NOT REST with httpx. Two client layers: HarnessServiceClientSync (benchmark orchestration) and PcmRuntimeClientSync (per-task file-system tools). Our httpx-based BitgnClient is entirely wrong. The tool surface is: tree, find, search, list, read, write, delete, mkdir, move, answer. Completion uses AnswerRequest with typed outcome codes. Python 3.14+ required, openai >=2.26 required. No API key needed for PAC1-DEV — runnable anonymously immediately.

## Affected Files

- `src/pac_cortex/client.py`
- `src/pac_cortex/config.py`
- `pyproject.toml`
