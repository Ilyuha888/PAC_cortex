# Three-layer safety: injection scan, tool allowlist, secret redaction

## Rationale

Safety module implements three defenses: (1) regex-based injection detection on tool results (system prompt override, role injection, instruction override, prompt leak, encoded payloads), (2) tool call validation against dynamic allowlist derived from task schemas, (3) secret redaction stripping api_key/secret/token/password patterns. These are scoring criteria for PAC challenge. Tests exist in test_safety.py.
