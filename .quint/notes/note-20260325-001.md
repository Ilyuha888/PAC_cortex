# create_autospec incompatible with Python 3.14 annotation evaluation (PEP 649)

## Rationale

create_autospec(VmClient, instance=True) crashes on Python 3.14.0 with TypeError: 'function' object is not subscriptable when inspect.signature tries to eagerly evaluate list[str] | None annotations under the new lazy-annotation regime (PEP 649 / annotationlib). Workaround: keep MagicMock(spec=VmClient). Kwarg drift on vm.answer is still caught at assertion time via assert_called_once_with. Re-evaluate if CPython fixes the regression or if we move to a Python version where create_autospec works.

## Affected Files

- `tests/conftest.py`
