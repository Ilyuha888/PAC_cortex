# Stateless per-task, sequential execution architecture

## Rationale

Each task gets a fresh agent loop — no state carried between tasks. Runner iterates tasks sequentially. No database, single process. Parallel execution noted as future optimization. Session orchestrator in runner.py manages fetch-solve-collect lifecycle with proper client cleanup.

## Affected Files

- `src/pac_cortex/runner.py`
- `src/pac_cortex/agent.py`
