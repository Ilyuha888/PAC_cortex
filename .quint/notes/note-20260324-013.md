# Gemini OAI-compat ignores JSON schema constraints on discriminated union tool field

## Rationale

First live run showed Gemini returning tool='ls', 'rmdir', 'cp' — Unix shell names, not PCM tool names. The beta.chat.completions.parse JSON schema is sent but not enforced by Gemini's OAI-compat shim. Fixed with: (1) explicit tool list in SYSTEM_PROMPT, (2) ValidationError catch + _SCHEMA_CORRECTION injection + retry up to _MAX_PARSE_RETRIES=2.
