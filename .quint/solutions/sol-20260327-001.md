# Solutions for: Model-internal reasoning tokens inflate token budget on simple structured calls

Problem: prob-20260327-002

## Variants (3)

### V1. V1 — extra_body passthrough in parse_step()

Add `extra_body: dict[str, Any] | None = None` to parse_step(). Pass it directly to the SDK call. Pre-flight call passes `extra_body={'thinking_config': {'thinking_budget': 0}}` (Gemini) or `extra_body={'reasoning_effort': 'low'}` (OpenAI o-series). Caller owns provider specifics.

**Strengths:**
- Zero abstraction — one line in parse_step, one kwarg at call site
- Provider-agnostic by design — llm.py has no branching
- Fully backward-compatible — extra_body=None by default

**Weakest link:** Provider-specific dict at call site breaks on model swap

**Risks:**
- Caller must know provider-specific dict shape
- Pre-flight in agent.py hardcodes Gemini key — breaks with OpenAI o-series swap

**Rollback:** Remove extra_body arg; revert agent.py pre-flight call

### V2. V2 — thinking_budget setting + parse_step() param

Add `LLM_THINKING_BUDGET: int | None` to settings (default None = uncontrolled). Add `thinking_budget: int | None = None` to parse_step(). Internally build `extra_body={'thinking_config': {'thinking_budget': N}}` when set. Pre-flight always passes thinking_budget=0. Main loop uses default (None = provider decides).

**Strengths:**
- Clean semantic name in code
- Default in settings lets ops control baseline reasoning budget
- Pre-flight suppression is explicit and self-documenting

**Weakest link:** thinking_config key is Gemini-specific — silent no-op on other providers

**Risks:**
- Still Gemini-shaped internally — thinking_config key won't work on OpenAI o-series
- Adds config surface for one parameter

**Rollback:** Remove thinking_budget from settings and parse_step(); revert agent.py call

### V3. V3 — extra_body passthrough + OPENAI_EXTRA_BODY_PREFLIGHT env var

Same extra_body passthrough as V1 but read the dict from a JSON env var `OPENAI_PREFLIGHT_EXTRA_BODY` at startup. Pre-flight always passes this env-configured dict. Default is `{'thinking_config': {'thinking_budget': 0}}` for Gemini.

**Strengths:**
- Provider swap just needs env var change — no code change
- Zero provider branching in llm.py or agent.py

**Weakest link:** JSON env var parsing errors are runtime-only, invisible in tests

**Risks:**
- JSON in env var is fragile and hard to validate
- Extra env complexity for a one-call-site fix
