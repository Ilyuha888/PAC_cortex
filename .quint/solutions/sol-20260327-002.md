# Solutions for: Agent deletes structural _* template files when told to "remove all"

Problem: prob-20260327-003

## Variants (2)

### V1. V1 — Prompt rule: _* files are structural scaffolds

Append one sentence to the DELETE RULE in _PROMPT_CORE: files starting with _ are structural scaffolds and must never be deleted under any instruction.

**Strengths:**
- Zero infrastructure change
- Consistent with how all other behavioral rules are expressed in this codebase
- Generalizes to any workspace using _-prefix convention

**Weakest link:** LLM compliance — prompt rules are advisory, not enforced

**Risks:**
- LLM compliance — model may not apply rule under adversarial or ambiguous phrasing

**Rollback:** Revert the one-line prompt addition.

### V2. V2 — Hard guard in validate_tool_call

In safety.py validate_tool_call(), block any delete call whose path basename starts with _.

**Strengths:**
- Enforced at code level — 100% reliable regardless of LLM reasoning
- No prompt bloat

**Weakest link:** Policy leakage — mixing behavioral convention into the security gate

**Risks:**
- Conflates convention-based scope with security policy in the safety gate
- Blocks legitimate deletion of _* files if ever needed

**Rollback:** Remove the added condition from validate_tool_call.
