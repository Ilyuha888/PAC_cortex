# Solutions for: Distill workflow writes card but omits required thread link

Problem: prob-20260327-007

## Variants (3)

### V1. V1 — Extend STRICT SCOPE distill bullet inline

Update the single 'distill' bullet in _PROMPT_CORE's STRICT SCOPE RULE to say: write to 02_distill/cards/ AND check 02_distill/threads/ for a matching thread. Append a backlink if a non-template thread exists; skip silently if none. One bullet edit, zero structural changes.

**Strengths:**
- Minimal blast radius — one bullet only
- Constraint respected: no thread creation, only update
- DECOMPOSE rule forces agent to plan both steps upfront
- No changes to pre-flight, AssembledPrompt, or other modules

**Weakest link:** Model must correctly judge topic similarity when selecting among multiple threads — wrong match is possible.

**Risks:**
- Topic matching is semantic, not structural — LLM may pick wrong thread

**Rollback:** Revert the single bullet in _PROMPT_CORE. No other files touched.

### V2. V2 — Separate DISTILL WORKFLOW bullet block

Keep STRICT SCOPE RULE unchanged. Add a new dedicated DISTILL WORKFLOW bullet after it that enumerates: (1) write card, (2) find thread, (3) append link. Explicit numbered sequence removes ambiguity.

**Strengths:**
- Explicit numbered sequence is unambiguous
- Separate section easier to extend later

**Weakest link:** Two rules now address 'distill' — STRICT SCOPE RULE still says 'only write to 02_distill/cards/' which contradicts the new block. Contradiction risk.

**Risks:**
- Rule contradiction between STRICT SCOPE and new block
- More prompt tokens

**Stepping stone:** yes — opens future possibilities

**Rollback:** Remove the new bullet block.

### V3. V3 — Conditional pre-flight section (_PROMPT_PKM)

Add a new _PROMPT_PKM section with full distill/thread rules, loaded by pre-flight assembler when PKM workspace is detected (02_distill/ present). Mirrors the entity-inbox pattern.

**Strengths:**
- Clean architectural separation
- Extensible for future PKM rules

**Weakest link:** Requires changes to _preflight, AssembledPrompt, _build_system_prompt, and _SYSTEM_PROMPT_FULL — large blast radius for a one-line rule gap.

**Risks:**
- Structural over-engineering for a one-bullet fix
- Pre-flight assembler reliability becomes critical path

**Stepping stone:** yes — opens future possibilities

**Rollback:** Remove _PROMPT_PKM, revert AssembledPrompt, _preflight, _build_system_prompt.
