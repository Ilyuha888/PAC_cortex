# Solutions for: Instruction hierarchy conflict resolution for AGENTS.md files

Problem: prob-20260327-010

## Variants (3)

### V1. A — Extend assembler: discover all AGENTS.md + hierarchy-aware output

Pre-flight finds all AGENTS.md files in tree (root + nested). Assembler LLM receives them annotated with paths and explicit hierarchy rules. Outputs conflict_detected, conflict_description, resolved agent_instructions (compatible with hierarchy). Main agent gets pre-resolved explicit rules — zero runtime reasoning required.

**Strengths:**
- Small-model-friendly: all conflict reasoning done in assembler, main agent follows explicit labels
- Single LLM call as today (just richer output)
- Maintains security: assembler still acts as injection firewall

**Weakest link:** Quality of the assembler LLM's semantic conflict analysis — a weaker model may miss subtle contradictions

**Risks:**
- Assembler prompt becomes complex — quality of conflict detection depends on assembler LLM
- VM call count grows by number of nested AGENTS.md files found

**Stepping stone:** yes — opens future possibilities

### V2. B — Runtime hierarchy: pass all AGENTS.md verbatim, agent resolves

Pre-flight discovers and reads all AGENTS.md files, but passes them verbatim (with path labels) to the main agent's system prompt. Main agent applies hierarchy rules at runtime using its own reasoning.

**Strengths:**
- Simpler pre-flight code
- Larger reasoning model can handle nuance

**Weakest link:** Runtime LLM reasoning capacity — fails for small models, violates stated constraint

**Risks:**
- Fails for smaller models (qwen3.5-27b) — explicitly excluded by constraint
- Injects raw file content into main context — security regression risk

### V3. C — Structural keyword matching: deterministic conflict detection

Programmatic pre-flight: parse root AGENTS.md for constraint keywords, check if nested AGENTS.md contains contradicting keywords (NOT, never, do NOT vs root's always/must). Flag conflict without LLM, include resolved output in prompt.

**Strengths:**
- Zero LLM calls for conflict detection
- Deterministic, fast, no model dependency

**Weakest link:** Lexical matching can't capture semantic conflicts — likely fails for real PAC tasks

**Risks:**
- Misses semantic conflicts — only catches surface-level keyword contradictions
- High false negative rate for nuanced conflicts like t21/t22
