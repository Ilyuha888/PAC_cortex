# Solutions for: NextStep schema lacks confidence and budget-awareness fields

Problem: prob-20260325-002

## Variants (3)

### V1. V1

Add confidence Literal field to NextStep; log it at debug level after each successful parse_step. No behavioral change.

**Strengths:**
- zero blast radius — no control flow changes
- structured signal for post-run analysis
- one field, one log line

**Weakest link:** LLM compliance with the new required field (could cause more schema retries)

**Risks:**
- LLM must learn new required field

**Rollback:** Remove field from NextStep and logger.debug line; remove confidence= from test fixtures

### V2. V2

Add confidence field AND use it to gate a retry: if confidence=low, inject a self-critique prompt and re-parse before executing.

**Strengths:**
- could improve correctness on uncertain steps

**Weakest link:** Unclear whether self-critique actually improves low-confidence steps

**Risks:**
- behavioral change — alters control flow
- adds API calls
- harder to test

### V3. V3

Skip the schema field; emit a separate heuristic confidence proxy (e.g. step number / stagnation counter ratio) to logs without touching NextStep.

**Strengths:**
- no schema change
- no LLM compliance risk

**Weakest link:** Heuristic proxy is a poor substitute for model-reported confidence

**Risks:**
- proxy metric is noisy
- doesn't capture model's actual uncertainty
