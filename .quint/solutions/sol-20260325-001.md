# Solutions for: runner.py silently swallows task-level exceptions

Problem: prob-20260324-010

## Variants (3)

### V1. V1: Narrow the catch

Replace bare except Exception with except (openai.APIError, ConnectError). Logic bugs propagate as crashes; only infra errors are swallowed.

**Strengths:**
- Logic bugs visible as crashes
- Minimal code delta
- Contract clear: infra = continue, bugs = stop

**Weakest link:** Completeness of the exception allowlist — a new infra exception type would fall through as a bug crash

**Risks:**
- New exception types not in the allowlist silently swallow

### V2. V2: Re-raise after logging

Keep broad catch, immediately re-raise. First task failure aborts entire session.

**Strengths:**
- Every exception is visible

**Weakest link:** Multi-task resilience: loses the constraint entirely

**Risks:**
- One bad task kills all remaining tasks

### V3. V3: Typed TaskResult + threshold abort

Wrap each task outcome in TaskResult(ok, error). Catch broadly, record. Abort if N consecutive failures.

**Strengths:**
- Structured per-task outcomes
- Configurable resilience threshold

**Weakest link:** Complexity cost disproportionate to a tactical fix

**Risks:**
- Premature abstraction for current scope

## Summary
