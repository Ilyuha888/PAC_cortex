# Solutions for: API call budget is undocumented and unenforced in code

Problem: prob-20260324-011

## Variants (3)

### V1. V1: Local counter in agent.py

Task-scoped int in solve_task(), incremented at every LLM and VM call site. No new classes, no shared state.

**Strengths:**
- Zero new abstractions
- Trivially auditable
- Works within existing function scope

**Weakest link:** Manual discipline required at each new call site

**Risks:**
- Must remember to increment at every future call site

### V2. V2: call_count on LLMClient + VmClient

Add .call_count attribute to both clients; budget check reads sum of both counts.

**Strengths:**
- Inspectable from outside solve_task
- Centralized per client

**Weakest link:** Task-boundary reset is non-trivial when clients are shared across tasks

**Risks:**
- Session vs. task scoping requires reset logic
- Requires changes to two client classes

### V3. V3: Shared BudgetGuard object

Dedicated BudgetGuard(limit) object passed into solve_task(); wraps increment + check in one interface.

**Strengths:**
- Clean interface
- Testable in isolation
- Enforces the pattern

**Weakest link:** Premature abstraction for a single call site

**Risks:**
- Overkill for current scale
- Adds a new class with no other users

## Summary
