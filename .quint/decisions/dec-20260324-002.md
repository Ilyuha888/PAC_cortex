# A: MagicMock(spec=...)

## 1. Problem Frame

**Signal:** tests/ has no conftest.py and no mock factories for VmClient or LLMClient. Every agent/runner test requires live SDK calls, making isolated unit testing impossible.

**Constraints:**
- Must not require live SDK/network for unit tests
- Mocks must faithfully reproduce SDK return shapes (dict[str,Any])

**Acceptance:** conftest.py exists with mock_vm_client and mock_llm_client fixtures; at least one agent or runner test uses them and passes with no network calls

## 2. Decision

**Selected:** A: MagicMock(spec=...)

Minimal code (~25 lines), zero new dependencies, catches attribute-name drift via spec, and return shape fidelity is sufficient for unit-testing solve_task(). Interface is small (10 VmClient methods, 1 LLMClient method) and changes rarely.

**Invariants:**
- No live network calls in any unit test
- Fixtures must faithfully return dict[str, Any] shapes (not bare MagicMock objects)

**Pre-conditions:**
- [ ] uv sync passes
- [ ] tests/__init__.py already exists

**Post-conditions:**
- [ ] tests/conftest.py exists with mock_vm_client and mock_llm_client fixtures
- [ ] tests/test_agent.py exists with >=1 passing test using both fixtures
- [ ] uv run pytest passes with no env vars set

**Admissibility:**
- NOT: Must not import or instantiate real VmClient or LLMClient in tests
- NOT: Must not add pytest-mock or any new non-stdlib test dependency

## 3. Rationale

| Variant | Verdict | Reason |
|---------|---------|--------|
| A: MagicMock(spec=...) | **Selected** | Minimal code (~25 lines), zero new dependencies, catches ... |
| B: Manual FakeVmClient / FakeLLMClient stubs | Rejected | 3-4x more code with high manual sync burden. Full signature checking is not worth the maintenance cost at this stage — the real interface is small and unlikely to change often. |
| C: pytest-mock mocker fixture | Rejected | Functionally identical to A but adds pytest-mock as a dependency with no meaningful gain. Auto-teardown is irrelevant when mocks are already fixture-scoped. |

**Weakest link:** Method signature drift (wrong kwargs) is not caught by MagicMock(spec=...) — only attribute name changes are detected

**Evidence requirements:**
- uv run pytest passes with no env vars
- uv run ruff check src/ tests/ passes
