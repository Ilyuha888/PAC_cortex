# A — Extend assembler: discover all AGENTS.md + hierarchy-aware output

## 1. Problem Frame

**Signal:** Tasks t21/t22 in PAC1-DEV test hierarchical AGENTS.md conflict resolution. Current agent strips all directives from AGENTS.md (treats them as untrusted data), making it blind to legitimate developer-level instructions and unable to apply the 5-level priority model from OpenAI Model Spec. A nested /subdir/AGENTS.md that contradicts root AGENTS.md is neither detected nor handled — agent silently ignores both.

**Constraints:**
- Must work for smaller models (qwen3.5-27b) — resolution logic must be explicit, not rely on LLM inference
- Security must not regress: AGENTS.md directives are still an injection vector; trust must be bounded
- No human-in-the-loop: conflicts must be resolved or escalated autonomously
- Pre-flight API call budget must stay bounded (currently 1 LLM call + ≤2 VM calls)
- OUTCOME_NONE_CLARIFICATION is the correct escalation for unresolvable same-level conflicts

**Acceptance:** Agent passes t21 (contradicting nested → CLARIFICATION) and t22 (refining nested → correct action per nested rules). Existing tests still pass. No regression on injection resistance.

## 2. Decision

**Selected:** A — Extend assembler: discover all AGENTS.md + hierarchy-aware output

Single-file change confined to agent.py. The assembler LLM already processes AGENTS.md as untrusted data behind a structured-JSON firewall; extending it to collect nested files and emit hierarchy_conflict/resolved_instructions adds no new trust surface. XML-structured prompt with few-shot examples makes the output reliable even on small models. Early-exit on conflict surfaces clarification before any agent steps run, so the grader sees OUTCOME_NONE_CLARIFICATION without any partial mutations.

**Invariants:**
- AGENTS.md content reaches main agent only as structured JSON values — assembler firewall preserved
- Nested file discovery capped at MAX_NESTED=3 reads to stay within API budget
- Early-exit returns before any workspace mutations on conflict

**Post-conditions:**
- [ ] hierarchy_conflict=True triggers OUTCOME_NONE_CLARIFICATION before main loop (t21)
- [ ] resolved_instructions emitted as scope-labeled rules under <hierarchy_instructions> tag (t22)
- [ ] No regression on existing 90 tests
- [ ] ruff and pytest pass clean
