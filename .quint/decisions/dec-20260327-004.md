# V1 — Prompt rule: _* files are structural scaffolds

## 1. Problem Frame

**Signal:** t01: "Remove all captured cards and threads" caused deletion of _card-template.md and _thread-template.md. Agent interprets "all files" literally with no concept that _-prefixed files are scaffolding.

**Constraints:**
- fix must not require per-workspace hardcoding
- must work for any workspace that uses _-prefix convention

**Acceptance:** t01 passes: template files survive a "remove all" instruction

## 2. Decision

**Selected:** V1 — Prompt rule: _* files are structural scaffolds

Behavioral scope miss (agent deleting scaffolds on "remove all") should be fixed at the prompt layer, not in the security gate. V1 is minimal (1 line), zero infrastructure change, and consistent with how all other behavioral rules are expressed in this codebase. Acceptable risk for tactical mode since LLM compliance is high on straightforward prompt rules like scaffold protection.

**Invariants:**
- _-prefix rule is prompt-only; no code-level enforcement
- DELETE RULE block remains intact — only two lines added after the existing rule
- No per-workspace hardcoding; rule applies universally

**Pre-conditions:**
- [ ] _PROMPT_CORE DELETE RULE block exists and is accessible to LLM during agent loop
- [ ] Test suite is clean (88/88 passing)

**Post-conditions:**
- [ ] Two new lines appended to DELETE RULE block in agent.py
- [ ] Ruff and pytest pass with no regressions
- [ ] t01 task trace shows _card-template.md and _thread-template.md survive
