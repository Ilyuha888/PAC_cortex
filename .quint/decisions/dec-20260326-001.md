# V2 — Targeted tests for client.py + llm.py

## 1. Problem Frame

**Signal:** Only tests/test_safety.py exists. Agent loop (solve_task), LLM client, BitGN client, runner, and CLI have zero tests. Agent loop is the most critical path and has complex branching (text response, tool calls, stagnation, budget exhaustion, injection detection).

**Constraints:**
- Tests must not require live API connections
- Must use pytest-asyncio for async code

**Acceptance:** Agent loop has tests for all 4 exit paths (text answer, stagnation, budget, disallowed tool). LLM and BitGN clients have mock-based tests.

## 2. Decision

**Selected:** V2 — Targeted tests for client.py + llm.py

client.py and llm.py had 0% coverage over non-trivial logic: VmClient._call() retry/timeout/ConnectError paths and LLMClient.parse_step() rate-limit backoff/token accumulation. These directly affect tool dispatch correctness and budget enforcement — both scored dimensions. V2 covers them with 21 focused tests in ~160 lines.

**Invariants:**
- No mocking of actual proto message classes — real AnswerRequest/ReadRequest used
- MagicMock(spec=...) pattern preserved for agent-layer fixtures in conftest.py
- Settings patched at pac_cortex.llm.settings and pac_cortex.client._settings to avoid env dependency

**Post-conditions:**
- [ ] 77 tests passing (56 before)
- [ ] ruff clean
- [ ] client.py retry/timeout/ConnectError/outcome-lookup paths covered
- [ ] llm.py token accumulation, rate-limit retry, exhaustion, and unknown-error propagation covered
