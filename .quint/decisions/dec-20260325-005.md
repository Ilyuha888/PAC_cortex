# V1

## 1. Problem Frame

**Signal:** NextStep provides current_state and plan_remaining_steps_brief — reasoning is structured and auditable. But the schema has no confidence estimate and no budget-awareness field. The LLM cannot signal uncertainty or self-regulate based on remaining step budget. These gaps become visible when an agent confidently executes the wrong plan for 10+ steps, or exhausts budget without noticing it's running low.

**Constraints:**
- Fields must not inflate token count significantly — each extra field costs prompt tokens on every step
- Must not require asyncio (stack is sync)
- Must degrade gracefully — LLM filling confidence=0.9 on wrong path is worse than not having it if not used for control flow

**Acceptance:** NextStep includes confidence: float (0-1) and steps_remaining: int fields; agent logs low-confidence steps; empirical check on ≥5 tasks shows no regression in token cost vs benefit

## 2. Decision

**Selected:** V1

Adds structured observability (confidence in structured logs) with zero behavioral change risk and minimum schema overhead (one field). Pareto-dominant over V2 (same field count, no control flow risk) and V3 (real model signal vs heuristic proxy).

**Invariants:**
- All existing tests pass
- Agent control flow unchanged

**Post-conditions:**
- [ ] confidence field present in NextStep schema
- [ ] logger.debug logs confidence after each successful parse_step
