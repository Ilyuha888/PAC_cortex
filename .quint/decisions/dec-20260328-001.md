# D — Assembler contract + lightweight step-level tracking

## 1. Problem Frame

**Signal:** Two failure modes in benchmark run (t07, t19) share a structural cause: safety invariants live as prose text in the system prompt and are enforced only by LLM persuasion at inference time. (1) Ordering: agent matches the first applicable exit condition and short-circuits past later checks (t07 — INBOX SINGULAR fires before filename scan). (2) Prior override: LLM pre-training priors override weak rule statements (t19 — email processing prior = read+act+delete overrides 'leave inbox files in place').

**Constraints:**
- No changes to test assertions
- No regression on existing 90 unit tests
- Assembler firewall preserved — AGENTS.md content reaches main agent only as structured JSON
- Budget per task stays under 1000 API calls
- Sequential task execution (no parallel)
- Stateless per-task — no cross-task memory

**Acceptance:** 1. t19-class failures impossible — code blocks inbox deletion without explicit authorization, regardless of LLM output. 2. t07-class failures addressed — required pre-checks enforced before ambiguity resolution. 3. All 90 unit tests pass. 4. Benchmark score >= 18/20 (excluding infra failures).

## 2. Decision

**Selected:** D — Assembler contract + lightweight step-level tracking

Two-layer enforcement: assembler produces task_contract (what's allowed), NextStep.checks_completed tracks execution progress (what's been done), dispatch code validates both before executing any tool call. Moves safety invariants from LLM persuasion to Python code. Graceful degradation: assembler failure → maximum restriction defaults. Contract violation → retry with correction, not abort.

**Invariants:**
- TaskContract defaults to maximum restriction (all false/empty) — safe direction on assembler failure
- Contract violations are retries not aborts — agent gets correction message and replans
- Assembler firewall preserved — AGENTS.md content still reaches main agent only as structured JSON
- ReqDelete on inbox/* is physically blocked by code when inbox_delete_authorized=False

**Pre-conditions:**
- [ ] Current prompt refactoring applied (this session's _PROMPT_* changes)
- [ ] All 90 unit tests passing
- [ ] Assembler prompt functional (pre-flight works)

**Post-conditions:**
- [ ] TaskContract model on AssembledPrompt with 3 fields
- [ ] checks_completed field on NextStep
- [ ] _enforce_contract() validates every tool call before dispatch
- [ ] Contract violation injects correction into conversation log
- [ ] Assembler prompt extended with task_contract output spec
- [ ] Prose rules softened: load-bearing sentences removed, replaced with guidance
- [ ] _MAX_STEPS raised to 50
- [ ] New unit tests: contract blocks inbox delete, blocks premature NONE_CLARIFICATION, blocks inbox read without filename_scan, fallback on assembler failure

**Admissibility:**
- NOT: No silent pass-through of contract violations
- NOT: No abort on contract violation (retry only)
- NOT: No new external dependencies
- NOT: No changes to VmClient or LLMClient interfaces

## 3. Rationale

| Variant | Verdict | Reason |
|---------|---------|--------|
| D — Assembler contract + lightweight step-level tracking | **Selected** | Two-layer enforcement: assembler produces task_contract (... |
| A — Execution contract only | Rejected | Handles t19 (inbox delete blocking) but insufficient for t07 (ordering). No mechanism to enforce filename scan before NONE_CLARIFICATION. D subsumes A. |
| B — Two-phase planning | Rejected | Adds 1 API call per task with plan-execution drift risk. Dominated by D which achieves same enforcement without extra call. |
| C — Stateful NextStep only | Rejected | LLM self-reporting is the weakest enforcement layer. Agent can declare checks complete without doing them. D uses C's tracking as secondary signal but relies on contract for critical invariants. |

**Weakest link:** Assembler LLM classification accuracy for task_contract fields. Mitigated by safe defaults (maximum restriction) and graceful retry on violation.

**Evidence requirements:**
- All 90+ unit tests pass (including new contract tests)
- Benchmark run: t19 passes (inbox file preserved)
- Benchmark run: t07 improves (filename scan forced before NONE_CLARIFICATION)
- Benchmark score >= 18/20 excluding infra failures
- Trace inspection: contract violation messages visible in traces for blocked actions

## 4. Consequences

**Rollback plan:**

2. Remove TaskContract from AssembledPrompt
3. Remove checks_completed from NextStep
4. Restore prose rules from git history
Blast radius: agent.py only — no external interfaces affected
