# TaskTracer class in tracer.py

## 1. Problem Frame

**Signal:** Running the benchmark produces no structured record of what the agent did per step. After a run you can see the final score but not: which NextStep the LLM chose at each step, which tools were called and with what args, what the tool returned, why the agent stopped (step limit / stagnation / answer). Debugging wrong answers or unexpected behavior requires re-running with added print statements.

**Constraints:**
- Trace must not leak secrets or raw env vars
- Must not add significant latency to the benchmark run
- Must survive a crash — trace must be flushed incrementally, not only at task end
- Must work without changing the agent's decision logic

**Acceptance:** After uv run python -m pac_cortex run, a per-task trace file exists showing the full step-by-step Q/A and tool history; readable without extra tooling; secrets are redacted

## 2. Decision

**Selected:** TaskTracer class in tracer.py

Thin dedicated class with incremental line-buffered writes. Crash-safe by design (flush after every write), fully optional (tracer=None), and readable without tooling. Passes all 54 tests with zero behavioral change to agent logic.

**Invariants:**
- Agent control flow unchanged
- All existing tests pass
- Tool results written only after redact_secrets()

**Post-conditions:**
- [ ] traces/ directory created at session start
- [ ] One .txt file per task per run
- [ ] Every early-exit path records to trace
- [ ] No secrets appear in trace files

## 3. Rationale

| Variant | Verdict | Reason |
