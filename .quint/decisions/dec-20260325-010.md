# A: Strict Gate (Fail-Fast)

## 1. Problem Frame

**Signal:** 3 concrete gaps confirmed by code inspection: (1) validate_tool_call() is dead code — written in safety.py:123-130, never imported/called in agent.py; path traversal in tool args goes unchecked. (2) No circuit breaker — injection warnings log and prepend a header but never abort; agent can process unlimited injected files. (3) Security outcome is advisory — agent relies on LLM calling OUTCOME_DENIED_SECURITY; nothing enforces it deterministically.

**Constraints:**
- No new dependencies
- safety.py API unchanged
- ~15 net new lines across 2 files
- All existing tests must pass

**Acceptance:** validate_tool_call imported and called before _dispatch; injection scan aborts task with OUTCOME_DENIED_SECURITY instead of prepending header; 3 new tests pass; all existing tests pass; ruff clean

## 2. Decision

**Selected:** A: Strict Gate (Fail-Fast)

Deterministic Python-layer abort satisfies binary grader scoring. Minimal surface (~15 lines, 2 files) minimizes April 11 deadline risk. validate_tool_call API already correct — no safety.py changes needed. Existing 22+ patterns have tested FP carve-outs.

**Invariants:**
- validate_tool_call called before every _dispatch
- injection abort uses vm.answer(OUTCOME_DENIED_SECURITY) not log-and-continue
- redact_secrets runs before abort check so no secrets are logged
- all existing tests pass

**Post-conditions:**
- [ ] validate_tool_call imported in agent.py
- [ ] _ALLOWED_TOOLS frozenset defined
- [ ] injection scan aborts deterministically
- [ ] 3 new tests pass: path_traversal, injection_abort, unknown_tool
