# Prompt-only fix: search tool + limit + duplicate detection + outbox schema

## 1. Problem Frame

**Signal:** t12: "Send follow-up to Alex Meyer" — agent found a contact and sent without checking uniqueness; expected OUTCOME_NONE_CLARIFICATION. t14: "Send to Blue Harbor Bank" — agent used company name as `to` field instead of looking up the email address.

**Constraints:**
- to field must always be a valid email address
- ambiguous contacts must trigger clarification, not a guess

**Acceptance:** t12 passes (clarification returned for ambiguous contact) AND t14 passes (email address resolved from account lookup)

## 2. Decision

**Selected:** Prompt-only fix: search tool + limit + duplicate detection + outbox schema

All gaps were in _PROMPT_ENTITY_INBOX. Three targeted additions: (1) mandate search over find with entity-ID explanation, (2) require limit>=5 so duplicate contacts are visible, (3) inline outbox schema requiring a valid email address resolved from entity file. No code changes needed.

**Invariants:**
- 90 tests pass
- ruff clean
- t12 returns OUTCOME_NONE_CLARIFICATION
- t14 returns OUTCOME_OK with resolved email address

## 3. Rationale

| Variant | Verdict | Reason |

| Code-level dedup guard in dispatch | Rejected | Would require post-hoc result parsing; prompt fix is simpler and catches the problem before tool execution |
| Pre-flight injects contact list | Rejected | Over-engineering for edge case; pre-flight tree is already expensive |
