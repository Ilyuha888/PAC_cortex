# Targeted unit tests for 4 missing agent.py control paths

## 1. Problem Frame

**Signal:** agent.py is 285 LoC and the most critical module. Step limit enforcement, stagnation detection, tool dispatch, injection integration, and schema parse retry are all untested — invisible until competition day.

**Constraints:**
- Tests must be deterministic — no live LLM or SDK calls
- Must cover the nominal path plus at least: step limit hit, stagnation abort, injection rejection

**Acceptance:** tests/test_agent.py exists; covers step-limit, stagnation, tool dispatch, and injection rejection paths; all pass with uv run pytest

## 2. Decision

**Selected:** Targeted unit tests for 4 missing agent.py control paths

All four critical paths (step budget exhausted, injection warning, schema retry exhausted, schema retry success) are now covered with fast, deterministic, mocked tests. Acceptance criteria met: 6 tests pass, ruff clean, no live calls.

**Invariants:**
- No live network calls in any unit test
- MagicMock fixtures return dict[str, Any] shapes, never bare MagicMock objects

**Pre-conditions:**
- [ ] conftest.py with mock_vm_client and mock_llm_client fixtures exists
- [ ] uv sync passes

**Post-conditions:**
- [ ] tests/test_agent.py has ≥6 passing tests
- [ ] step-limit, stagnation, tool dispatch, injection rejection, schema retry exhausted, schema retry success all covered
- [ ] uv run ruff check src/ tests/ passes
- [ ] uv run pytest passes

**Admissibility:**
- NOT: NOT: tests that import or instantiate real VmClient or LLMClient
- NOT: NOT: tests that depend on env vars or network
- NOT: NOT: kwarg-level assertion drift — MagicMock(spec=) does not catch wrong kwargs

## 3. Rationale

| Variant | Verdict | Reason |
|---------|---------|--------|
| Targeted unit tests for 4 missing agent.py control paths | **Selected** | All four critical paths (step budget exhausted, injection... |
| Integration tests with live LLM/SDK | Rejected | Violates constraint: tests must be deterministic with no live calls. Slow and non-reproducible in CI. |
| No additional tests (leave at 2) | Rejected | Step budget and injection paths — the most likely failure modes on competition day — remained untested. |

**Weakest link:** MagicMock(spec=VmClient) enforces attribute names only — wrong kwargs on vm.answer/vm.read etc. are silently accepted, so kwarg drift between agent.py and VmClient is not caught by these tests

**Evidence requirements:**
- uv run pytest passes with 0 failures
- uv run ruff check src/ tests/ passes

## 4. Consequences

**Rollback plan:**
Triggers:
- conftest.py change breaks test_safety.py
- agent.py interface refactor invalidates test assertions
Steps:
1. Delete added tests from test_agent.py
2. Revert conftest.py if modified
Blast radius: local test files only

**Refresh triggers:**
- agent.py control flow changes (new paths, changed budget constants)
- VmClient.answer signature changes (kwargs drift not caught by spec)
- LLMClient.parse_step signature changes

**Affected files:** tests/test_agent.py, tests/conftest.py
