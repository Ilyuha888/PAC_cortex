# Task-specific entity link injection in pre-flight

## 1. Problem Frame

**Signal:** t13: "Nordlicht Health asked to reconnect in two weeks" — agent updated rem_001.json but missed the required write to accounts/acct_001.json next_followup field. The link between reminders/ and accounts/ isn't in the prompt.

**Constraints:**
- fix must generalize to other cross-resource relationships, not just reminders+accounts

**Acceptance:** t13 passes: both rem_001.json and acct_001.json updated on reschedule

## 2. Decision

**Selected:** Task-specific entity link injection in pre-flight

Deterministic VM lookups (no LLM) during pre-flight find the relevant source file, read linked entity files, detect mirrored date fields, and inject a concrete imperative note into the system prompt. Outperforms the generic ENTITY REFS prompt rule because it names the actual file and field, and explicitly demands a `context` call for date arithmetic.

**Invariants:**
- Safety pipeline unchanged — no LLM sees raw VM results during discovery
- Budget accounting: +2 calls (list/search + read) only when a note is produced
- Non-scheduling tasks: zero extra VM calls (_SCHEDULING_KEYWORDS gate)

**Post-conditions:**
- [ ] Entity link note names specific source path, linked file, and mirror field(s)
- [ ] Note demands `context` call before date arithmetic
- [ ] Discovery runs regardless of include_entity_inbox classification
- [ ] Pre-flight falls back gracefully: entity links appended to full prompt if assembler fails
- [ ] Assembler token limit raised to 8192 to reduce fallback rate

## 3. Rationale

| Variant | Verdict | Reason |
|---------|---------|--------|
| Task-specific entity link injection in pre-flight | **Selected** | Deterministic VM lookups (no LLM) during pre-flight find ... |
| Strengthen ENTITY REFS prompt rule wording | Rejected | Generic language ('may contain fields…') is still skippable when STRICT SCOPE signal is present; agent has no grounded file/field reference to act on. |
| Assembler-based link discovery | Rejected | Assembler already misclassifies scheduling tasks as entity_inbox=False; chaining discovery on that gate perpetuates the same failure. |

**Evidence requirements:**
- t13 score = 1.00 after fix
- Non-scheduling tasks produce no entity link note (regression check)

## 4. Consequences

**Affected files:** src/pac_cortex/agent.py
