# Add pre-flight trace visibility + strengthen contact search resilience

## 1. Problem Frame

**Signal:** t17: search("Brouwer Marit") returned empty, agent reported OUTCOME_NONE_CLARIFICATION without attempting list+scan fallback — even though a CONTACT LOOKUP FALLBACK rule exists in the prompt. t12 applied the same recovery by intuition when find() failed. Rules are present but not reliably triggered on matching conditions.

**Constraints:**
- No external LLM calls beyond the existing pre-flight + SGR loop budget
- NextStep schema changes require updating all test instantiations (~10 call sites)
- Must not regress passing tasks (currently 19/20 in vm-02khb run)

**Acceptance:** t17 passes consistently; no regressions on t01–t16, t18–t20; named rules appear in traces when their trigger fires

## 2. Decision

**Selected:** Add pre-flight trace visibility + strengthen contact search resilience

Addresses both sub-problems: (A) pre-flight is now recorded in every trace file via record_preflight(), making entity_inbox/vocab decisions observable post-hoc; (B) CONTACT SEARCH rule now requires last-name + first-name retries before giving up, eliminating the t17 one-attempt failure pattern.

**Invariants:**
- _preflight() falls back to _SYSTEM_PROMPT_FULL on any error — no partial prompts
- record_preflight() is called before the agent loop regardless of pre-flight success/failure
- Search resilience rule does NOT change outcome codes — only delays OUTCOME_NONE_CLARIFICATION until alternatives exhausted

**Post-conditions:**
- [ ] Every trace file opens with a Pre-flight: line
- [ ] _preflight() returns tuple[str, int, AssembledPrompt | None]
- [ ] CONTACT SEARCH rule present in _PROMPT_ENTITY_INBOX

## 3. Rationale

| Variant | Verdict | Reason |
|---------|---------|--------|
| Add pre-flight trace visibility + strengthen contact search resilience | **Selected** | Addresses both sub-problems: (A) pre-flight is now record... |
| Defer trace visibility | Rejected | Black-box pre-flight makes debugging future misclassifications impossible; cost is one method addition. |
| Hardcode search retries in agent loop code | Rejected | Prompt-level rule is sufficient and avoids baking search logic into the dispatch path; keeps the agent loop generic. |

**Evidence requirements:**
- 19/19 tests pass
- ruff clean
- trace file contains Pre-flight: line

**Affected files:** src/pac_cortex/agent.py, src/pac_cortex/tracer.py, tests/test_agent.py
