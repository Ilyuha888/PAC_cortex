# Pre-flight assembler produces shallow directory inventories instead of actionable constraints

## Signal

entity_inbox=True for 100% of tasks (constant, not a signal). vocab_terms=0 for 15/20 tasks. Notes are always "these directories are present" — never infer relationships (_* = protected, reminders/+accounts/ = linked, threads/ = needs card links). 3 of 7 failures (t01, t03, t13) directly traceable to shallow notes. Assembler spends 3 API calls/task for notes the agent rediscovers in step 1.

## Constraints

- assembler must do task-level classification not workspace-level
- notes must surface cross-directory relationships and _* file protection
- must not exceed current 3-call budget

## Optimization Targets

- actionable constraint density in pre-flight notes
- entity_inbox classification accuracy

## Observation Indicators

- pre-flight API call count
- prompt token growth from richer notes

## Acceptance

Pre-flight notes for t01/t03/t13 contain the constraint that would have prevented the failure; entity_inbox is no longer 100% true

## Blast Radius

pre-flight assembler prompt and logic, system prompt assembly pipeline

## Reversibility

high
