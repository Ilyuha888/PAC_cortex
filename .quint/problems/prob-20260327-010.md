# Instruction hierarchy conflict resolution for AGENTS.md files

## Signal

Tasks t21/t22 in PAC1-DEV test hierarchical AGENTS.md conflict resolution. Current agent strips all directives from AGENTS.md (treats them as untrusted data), making it blind to legitimate developer-level instructions and unable to apply the 5-level priority model from OpenAI Model Spec. A nested /subdir/AGENTS.md that contradicts root AGENTS.md is neither detected nor handled — agent silently ignores both.

## Constraints

- Must work for smaller models (qwen3.5-27b) — resolution logic must be explicit, not rely on LLM inference
- Security must not regress: AGENTS.md directives are still an injection vector; trust must be bounded
- No human-in-the-loop: conflicts must be resolved or escalated autonomously
- Pre-flight API call budget must stay bounded (currently 1 LLM call + ≤2 VM calls)
- OUTCOME_NONE_CLARIFICATION is the correct escalation for unresolvable same-level conflicts

## Optimization Targets

- Agent correctly follows nested AGENTS.md as local refinement when it doesn't violate root
- Agent returns OUTCOME_NONE_CLARIFICATION when nested AGENTS.md contradicts a root constraint
- Resolved instructions surface in system prompt as explicit, labeled rules (not background context)

## Observation Indicators

- Pre-flight LLM call count — must not exceed 2 total
- System prompt token count — watch for bloat from verbose instruction sets

## Acceptance

Agent passes t21 (contradicting nested → CLARIFICATION) and t22 (refining nested → correct action per nested rules). Existing tests still pass. No regression on injection resistance.

## Blast Radius

src/pac_cortex/agent.py — preflight, assembler prompt, AssembledPrompt model, system prompt builder. No runtime loop changes needed.

## Reversibility

high
