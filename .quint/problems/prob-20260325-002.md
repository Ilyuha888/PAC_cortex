# NextStep schema lacks confidence and budget-awareness fields

## Signal

NextStep provides current_state and plan_remaining_steps_brief — reasoning is structured and auditable. But the schema has no confidence estimate and no budget-awareness field. The LLM cannot signal uncertainty or self-regulate based on remaining step budget. These gaps become visible when an agent confidently executes the wrong plan for 10+ steps, or exhausts budget without noticing it's running low.

## Constraints

- Fields must not inflate token count significantly — each extra field costs prompt tokens on every step
- Must not require asyncio (stack is sync)
- Must degrade gracefully — LLM filling confidence=0.9 on wrong path is worse than not having it if not used for control flow

## Optimization Targets

- Agent self-signals low confidence before burning further steps
- Step budget is visible inside the schema so LLM can pace tool calls

## Observation Indicators

- Token overhead per step vs baseline NextStep
- False confidence rate — high confidence on wrong-answer tasks

## Acceptance

NextStep includes confidence: float (0-1) and steps_remaining: int fields; agent logs low-confidence steps; empirical check on ≥5 tasks shows no regression in token cost vs benefit

## Blast Radius

agent.py (NextStep schema), llm.py (token tracking). No other modules affected.

## Reversibility

high
