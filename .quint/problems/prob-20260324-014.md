# System prompt has no design rationale documentation

## Signal

The 70-line SYSTEM_PROMPT in agent.py encodes critical behavioral rules — CAPTURING rule, STRICT SCOPE, FILENAME handling, suspicious file handling — with no accompanying explanation of why each rule exists. Tuning or debugging the prompt requires reverse-engineering intent from phrasing.

## Constraints

- Rationale doc must be in the repo, not only in Quint
- Must explain the threat model each rule defends against

## Optimization Targets

- Each SYSTEM_PROMPT rule has a documented rationale and threat model
- Prompt changes can be evaluated against documented intent

## Acceptance

docs/system-prompt-rationale.md (or inline comments) explains each major SYSTEM_PROMPT rule with its rationale and the threat/scenario it addresses

## Blast Radius

Any change to the system prompt risks silently breaking a constraint whose purpose is unknown. Debugging unexpected LLM behavior requires guessing which prompt rule applies.

## Reversibility

high
