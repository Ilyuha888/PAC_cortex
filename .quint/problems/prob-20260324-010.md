# runner.py silently swallows task-level exceptions

## Signal

runner.py:30-34 catches all exceptions with bare `except Exception: logger.exception(...)` and continues to the next task. A crashed task is indistinguishable from a zero-score task in aggregate results, making post-run debugging opaque.

## Constraints

- Must not crash the entire session when one task fails
- Must preserve enough context (task_id, exception type, traceback) for post-run diagnosis

## Optimization Targets

- Failed tasks are clearly distinguished from zero-score completions in run output
- Exception info is structured and queryable post-run

## Acceptance

runner.py records task failures with structured metadata; run summary distinguishes crashed vs completed tasks; existing behavior (continue after failure) is preserved

## Blast Radius

Silent failures hide bugs that affect score. In a competition run, a systematic crash in agent.py would go unnoticed until results are published.

## Reversibility

high
