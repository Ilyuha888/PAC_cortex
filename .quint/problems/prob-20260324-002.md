# Safety module needs hardening for competition scoring

## Signal

Current injection detection uses 5 regex patterns — likely insufficient for competition-grade adversarial inputs. No argument sanitization on tool calls (only name validation). Secret redaction only covers key=value patterns, not structured JSON secrets. No test coverage for edge cases like unicode obfuscation, nested injections, or partial matches.

## Constraints

- Must not introduce false positives that block legitimate tool use
- Must be fast — runs on every tool result in the hot loop

## Optimization Targets

- Injection detection recall
- Zero secret leakage

## Observation Indicators

- False positive rate on benign tool results
- Latency overhead per scan

## Acceptance

Passes adversarial test suite with >95% detection rate. No secrets leak in any test scenario. False positive rate <1% on benign corpus.

## Blast Radius

safety.py, agent.py, test_safety.py

## Reversibility

high
