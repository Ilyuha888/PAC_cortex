# BitGN API contract unknown — client is a stub

## Signal

BitgnClient has placeholder endpoints (/tasks, /tasks/{id}/runs, /runs/{id}/tools/{name}, /runs/{id}/submit) that are guesses. Real API contract depends on SDK drop. All Pydantic models are missing — currently using dict[str, Any] everywhere.

## Constraints

- Must not break existing agent loop contract
- Must use Pydantic models at API boundaries per project rules

## Optimization Targets

- Type safety at API boundaries
- Fast adaptation when SDK drops

## Acceptance

BitgnClient uses typed Pydantic request/response models matching real API. Agent loop consumes typed objects instead of raw dicts.

## Blast Radius

client.py, agent.py, runner.py, config.py

## Reversibility

high
