# No per-task agent trace — benchmark runs are a black box

## Signal

Running the benchmark produces no structured record of what the agent did per step. After a run you can see the final score but not: which NextStep the LLM chose at each step, which tools were called and with what args, what the tool returned, why the agent stopped (step limit / stagnation / answer). Debugging wrong answers or unexpected behavior requires re-running with added print statements.

## Constraints

- Trace must not leak secrets or raw env vars
- Must not add significant latency to the benchmark run
- Must survive a crash — trace must be flushed incrementally, not only at task end
- Must work without changing the agent's decision logic

## Optimization Targets

- Each agent step is recorded: LLM input summary, NextStep chosen, tool called + args, tool result, step index
- Trace is human-readable after a run without tooling

## Observation Indicators

- Trace file size per task
- Wall-clock overhead of trace writes

## Acceptance

After uv run python -m pac_cortex run, a per-task trace file exists showing the full step-by-step Q/A and tool history; readable without extra tooling; secrets are redacted

## Blast Radius

Affects all post-run analysis and pre-competition tuning. Without a trace, system-prompt changes and tool improvements are evaluated blind — only end score changes are observable.

## Reversibility

high
