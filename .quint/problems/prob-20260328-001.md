# Agent invariants enforced by prose rules — ordering bugs and prior override

## Signal

Two failure modes in benchmark run (t07, t19) share a structural cause: safety invariants live as prose text in the system prompt and are enforced only by LLM persuasion at inference time. (1) Ordering: agent matches the first applicable exit condition and short-circuits past later checks (t07 — INBOX SINGULAR fires before filename scan). (2) Prior override: LLM pre-training priors override weak rule statements (t19 — email processing prior = read+act+delete overrides 'leave inbox files in place').

## Constraints

- No changes to test assertions
- No regression on existing 90 unit tests
- Assembler firewall preserved — AGENTS.md content reaches main agent only as structured JSON
- Budget per task stays under 1000 API calls
- Sequential task execution (no parallel)
- Stateless per-task — no cross-task memory

## Optimization Targets

- Move safety invariants from prose persuasion to code enforcement (dispatch-level blocking)
- Reduce hardcoded per-failure prompt rules — generalize through assembler classification + schema enforcement
- Maintain or improve benchmark score (current: 18/22 = 81.8%, excluding 2 infra failures = 18/20 = 90%)

## Observation Indicators

- Token usage per task (don't over-optimize — budget is generous)
- Pre-flight API calls (currently 2-4, can grow but watch)
- False positive rate on dispatch blocking (over-blocking legitimate operations)

## Acceptance

1. t19-class failures impossible — code blocks inbox deletion without explicit authorization, regardless of LLM output. 2. t07-class failures addressed — required pre-checks enforced before ambiguity resolution. 3. All 90 unit tests pass. 4. Benchmark score >= 18/20 (excluding infra failures).

## Blast Radius

agent.py (prompt constants + dispatch + NextStep schema), safety.py (dispatch validation), possibly assembler prompt — no CI/migration/config changes

## Reversibility

high
