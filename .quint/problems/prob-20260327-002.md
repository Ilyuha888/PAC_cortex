# Model-internal reasoning tokens inflate token budget on simple structured calls

## Signal

Gemini 2.5 Flash consumes ~490 "reasoning tokens" internally before producing text output — even for the pre-flight assembler call that returns a tiny JSON (~50 tokens of actual output). This consumed the entire original max_completion_tokens=512 budget, causing pre-flight to fail silently. Workaround: bumped to 4096. Root question: can model-internal reasoning be disabled via API for deterministic/simple structured-output calls?

## Constraints

- Must work via OpenAI-compatible API endpoint (used for Gemini and other providers)
- Cannot change LLM provider mid-session
- Pre-flight assembler output is ~50 tokens of JSON — reasoning overhead is disproportionate

## Optimization Targets

- Reduce token overhead for simple structured calls (pre-flight assembler)
- Understand API surface: which parameters control model-internal thinking

## Observation Indicators

- Pre-flight reliability (must not regress — keep fallback path)
- Per-task API call budget consumption

## Acceptance

Pre-flight assembler call succeeds with lower max_completion_tokens OR reasoning tokens are suppressed; actual token usage confirmed in logs

## Blast Radius

llm.py parse_step call signature; pre-flight only — main SGR loop unaffected

## Reversibility

high — parameter change in one call site
