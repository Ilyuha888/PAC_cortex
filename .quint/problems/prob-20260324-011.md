# API call budget is undocumented and unenforced in code

## Signal

CLAUDE.md states ~1000 API calls/task budget. Code enforces 30 LLM steps (_MAX_STEPS=30 in agent.py:17) but does not count or cap total API calls (tool calls + LLM calls combined). README claims a "hard guard" that does not exist.

## Constraints

- Hard stop must trigger before exceeding the budget, not after
- Budget counter must be visible in run logs for diagnosis
- Must not break the existing _MAX_STEPS guard

## Optimization Targets

- Total API call count (LLM + tool) tracked per task
- Hard stop enforced before budget is exhausted

## Observation Indicators

- LLM steps used per task
- Tool calls per task

## Acceptance

agent.py tracks total_api_calls (LLM + tool), enforces a configurable hard cap, logs the count per task, and README budget claim matches implementation

## Blast Radius

Exceeding the competition budget cap could disqualify a task or incur scoring penalties. The current guard only limits LLM steps, not tool-heavy execution paths.

## Reversibility

medium
